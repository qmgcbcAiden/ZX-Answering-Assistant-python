# 系统托盘功能使用说明

## 功能概述

ZX答题助手在 Windows 上支持系统托盘功能，允许程序最小化到任务栏右下角的系统托盘，在后台运行。

## 安装依赖

Windows 系统托盘功能需要额外的依赖包：

```bash
pip install pystray Pillow
```

或者重新安装所有依赖：

```bash
pip install -r requirements.txt
```

## 功能特性

### 1. 最小化到托盘
- 启用后，点击窗口最小化按钮时，程序会隐藏到系统托盘
- 双击托盘图标或选择"显示窗口"可重新显示窗口

### 2. 关闭到托盘
- 启用后，点击窗口关闭按钮时，程序不会退出，而是隐藏到系统托盘
- 通过托盘菜单的"退出"选项才能真正关闭程序

### 3. 托盘菜单
右键点击托盘图标可以访问以下功能：
- **显示窗口** - 恢复程序窗口
- **隐藏窗口** - 隐藏程序窗口到托盘
- **退出** - 完全关闭程序

### 4. 提取任务后台运行
- 在答案提取进度对话框中点击"后台运行"，即使未长期启用托盘设置，也会临时创建托盘图标
- 从临时托盘恢复窗口后，未启用常驻设置的图标会自动移除

## 配置方法

1. 启动 ZX答题助手
2. 进入"系统设置"页面
3. 找到"窗口与托盘"部分
4. 根据需要启用以下选项：
   - **最小化到系统托盘** - 启用最小化到托盘功能
   - **关闭到系统托盘** - 启用关闭到托盘功能
5. 点击"保存设置"按钮
6. 设置立即生效，无需重启应用程序

## 测试功能

运行测试脚本验证托盘功能是否正常：

```bash
python test_tray.py
```

测试脚本会：
1. 检查依赖是否安装
2. 启动托盘图标
3. 显示操作提示
4. 等待用户通过托盘菜单测试各项功能

## 使用场景

### 适合启用托盘功能的场景：
- 需要长时间运行答题任务
- 希望程序在后台运行不影响其他工作
- 需要快速访问程序而不占用任务栏空间

### 不适合启用托盘功能的场景：
- 只是临时使用程序
- 需要频繁查看程序界面
- 不熟悉系统托盘操作

## 注意事项

1. **依赖要求**: 系统托盘功能需要安装 `pystray` 和 `Pillow` 包
2. **即时生效**: 修改托盘设置并保存后会立即启动或停止托盘图标
3. **完全退出**: 启用"关闭到托盘"后，需要通过托盘菜单的"退出"选项才能完全关闭程序
4. **故障保护**: 托盘图标未成功显示时，应用不会隐藏主窗口
5. **平台支持**: 当前托盘实现仅在 Windows 上启用；macOS/Linux 仍可运行应用，但不会启动托盘组件

## 故障排除

### 托盘功能不可用
- 检查是否安装了依赖: `pip install pystray Pillow`
- 查看设置页面的托盘状态指示

### 托盘图标不显示
- 确保系统托盘功能在操作系统中已启用
- 检查系统托盘设置是否隐藏了图标

### 无法恢复窗口
- 双击托盘图标
- 右键点击托盘图标选择"显示窗口"
- 如果仍然无法显示，重启程序

## 技术细节

- **图标位置**: 任务栏右下角系统托盘区域
- **图标文件**: `build/flutter/windows/runner/resources/app_icon.ico`
- **事件处理**: `pystray` 线程仅接收菜单动作，窗口操作会调度回 Flet 事件循环
- **实现方式**: 使用 `pystray` 提供系统托盘图标

## 配置文件

托盘相关设置随应用配置保存在用户配置目录：

- Windows: `%APPDATA%\ZX-Answering-Assistant\cli_config.json`
- macOS: `~/Library/Application Support/ZX-Answering-Assistant/cli_config.json`
- Linux: `${XDG_CONFIG_HOME:-~/.config}/ZX-Answering-Assistant/cli_config.json`

```json
{
  "gui_settings": {
    "minimize_to_tray": false,
    "close_to_tray": false
  }
}
```

建议通过程序界面进行设置。旧版 `src/cli_config.json` 会在首次启动时自动迁移。
