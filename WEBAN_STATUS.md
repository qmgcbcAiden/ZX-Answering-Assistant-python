## ⚠️ 当前限制

### 1. 手动作答暂不支持
**问题**：WeBan 原代码在遇到题库中没有答案的题目时，会使用 `input()` 等待用户输入。这在 CLI 模式下工作良好，但在 GUI 模式下会导致程序挂起。

**影响**：
- 如果题库中没有答案，GUI 版本可能无法正常工作
- 需要手动作答的题目会导致程序卡住

**临时解决方案**：
1. **使用 CLI 模式运行 WeBan**（推荐）：
   ```bash
   cd src/modules/WeBan
   python main.py
   ```
   这样可以完整使用所有功能，包括手动作答。

2. **确保题库完整**：
   - 使用已有完整题库的学校
   - 先在 CLI 模式下运行一次，完善题库

3. **等待后续版本**：
   - 下一版本将添加 GUI 输入对话框
   - 支持在 GUI 中手动作答

### 2. 验证码识别
- ✅ 支持：ddddocr 验证码识别
- ❌ 不支持：腾讯云验证码（部分学校使用）

## 🔧 技术细节

### 架构设计
```
GUI (Flet)
  ↓
WeBanView (weban_view.py)
  ↓
WeBanAdapter (weban_adapter.py)
  ↓
WeBanClient (WeBan/client.py)
  ↓
WeBanAPI (WeBan/api.py)
```

### 线程模型
- **主线程**：Flet UI 事件循环
- **工作线程**：WeBan 任务执行
- **线程安全**：使用 `page.update_threadsafe()` 更新 UI

### 已知的兼容性问题
1. **`input()` 在 GUI 中不可用**
   - WeBan 原代码使用 `input()` 获取用户输入
   - 需要改为回调函数或对话框方式
   - 计划在 v2.9.0 中添加

2. **`scroll_to()` 是协程**
   - 已移除该调用，避免 RuntimeWarning
   - 日志自动滚动功能暂时禁用

## 💡 使用建议

### 推荐使用场景
1. ✅ 题库完整的学校
2. ✅ 不需要验证码的学校
3. ✅ 只使用自动功能

### 不推荐使用场景
1. ❌ 需要频繁手动作答
2. ❌ 使用腾讯云验证码
3. ❌ 题库不完整

### 最佳实践
1. **首次使用**：先在 CLI 模式下测试，确保功能正常
2. **题库准备**：使用 CLI 模式完善题库
3. **定期同步**：定期运行 CLI 版本同步最新题库
4. **备选方案**：GUI 模式遇到问题时，切换到 CLI 模式
