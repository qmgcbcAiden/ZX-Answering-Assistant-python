# WeBan GUI 集成说明

## 🎯 功能概述

WeBan 模块已完整集成到 GUI 中，提供以下功能：

1. ✅ **学校验证** - 自动验证学校名称
2. ✅ **GUI 登录** - 图形界面登录 WeBan
3. ✅ **后台执行** - 任务在后台线程运行，不阻塞 UI
4. ✅ **交互弹窗** - 验证码、手动答题通过弹窗完成
5. ✅ **实时日志** - 控制台显示执行进度
6. ✅ **停止功能** - 可随时停止执行

## 🔧 使用方法

### 1. 启动程序
```bash
python main.py
```

### 2. 导航到 WeBan 模块
- 点击左侧导航栏的"安全微伴"图标

### 3. 配置登录信息
- **学校名称**：输入完整学校名（如：云南林业职业技术学院）
- **账号**：输入您的账号
- **密码**：输入您的密码

### 4. 验证学校（可选）
- 点击"验证学校"按钮
- 验证成功后会提示：✓ 学校验证成功

### 5. 开始执行
- 点击"登录并开始"按钮
- 控制台对话框自动弹出
- 点击"开始执行"按钮

## 💡 交互场景

### 场景 1：验证码输入
1. OCR 自动识别前 3 次
2. 失败后弹出对话框，显示验证码图片
3. 用户在弹窗中输入验证码
4. 点击"确认"提交

### 场景 2：重考选择
1. 如果考试已通过，弹出确认对话框
2. 显示历史成绩和剩余次数
3. 用户选择"是"或"否"

### 场景 3：手动答题
1. 题库中没有答案时，弹出答题对话框
2. 显示题目和选项
3. 用户输入答案序号（如：1,2,3）

## ⚙️ 技术细节

### 输入处理流程
```
WeBan 模块（后台线程）
  ↓ 需要用户输入
adapter.input_callback(prompt)
  ↓ 调用
WeBanView._handle_input(prompt)
  ↓ 显示对话框
WeBanInputDialog.show_*_dialog()
  ↓ 用户输入
返回结果
  ↓ 继续
WeBan 模块继续执行
```

### 线程安全
- 所有 UI 更新通过 `page.update()` 在主线程执行
- 对话框使用 `threading.Event` 等待用户输入
- 不阻塞主 UI 线程

### 打包兼容性
- ✅ 完全兼容 PyInstaller 打包
- ✅ 不依赖外部 Python 环境
- ✅ 不使用系统命令（start、taskkill 等）
- ✅ 真正的跨平台支持

## 🐛 常见问题

### Q1: 验证码无法识别
**A**:
- 前 3 次会自动 OCR 识别
- 失败后会弹出图片输入框
- 确保 `ddddocr` 已安装：`pip install ddddocr==1.6.1`

### Q2: 程序卡住不动
**A**:
- 检查是否有弹窗被遮挡
- 查看"任务管理器"是否有 Python 进程
- 点击"停止执行"尝试恢复

### Q3: 题库不完整
**A**:
- 程序会弹出对话框提示手动答题
- 完成后会继续自动答题
- 建议使用完整的题库

## 🎓 使用技巧

1. **提前验证学校** - 避免登录时才发现学校名称错误
2. **使用完整题库** - 减少手动答题次数
3. **耐心等待** - 学习和考试需要时间，请勿频繁停止
4. **查看日志** - 控制台会显示详细进度
5. **适时停止** - 发现问题及时点击"停止执行"
